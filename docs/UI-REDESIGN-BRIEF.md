# ТЗ для Claude Design — современный редизайн «Финанс-Элит»

> Документ-бриф для генерации макетов в Claude Design. Составлен так, чтобы подавать
> **поэтапно**: один «Этап» = один запрос. Каждый этап самодостаточен, но требует
> прикладывать **Глобальный контекст** (разделы 1–3). Цель — чтобы Claude Design на каждом
> этапе уходил вглубь и прорабатывал все состояния, адаптив и обе темы.
>
> ⚠️ **Главное требование заказчика:** весь существующий функционал должен сохраниться и
> быть выражен в новых макетах. Раздел 3 — чек-лист функционала; по нему сверяемся, что
> ничего не потеряно. После макетов мы будем связывать их с backend — поэтому в каждом
> экране указаны реальные данные/поля (имена, диапазоны, примеры).

---

## 0. Как пользоваться этим ТЗ

1. **Один этап = один запрос** в Claude Design. Не подавай всё сразу.
2. **Перед каждым этапом** прикладывай разделы **1 (Контекст), 2 (Дизайн-направление),
   и токены из Этапа 1 (дизайн-система)** — чтобы стиль был сквозным.
3. В конце каждого этапа есть блок **«Директива на проработку»** — копируй его, он
   заставляет дизайнера раскрыть все состояния и детали, а не отдать «один экранчик».
4. Порядок этапов: сначала **Этап 1 (дизайн-система)** — это фундамент. Затем экраны.
5. Для каждого экрана проси: **light + dark тему, desktop + tablet + mobile, все состояния
   (default / hover / focus / disabled / loading / empty / error), и спецификацию
   компонентов** (отступы, размеры, токены).

## 1. Контекст продукта (прикладывать к каждому этапу)

**Что это.** «Финанс-Элит» — веб-SaaS для финансового моделирования предприятия и
инвестпроектов (современный аналог Project Expert / Альт-Инвест). Пользователь вводит
модель бизнеса, а система считает помесячно на горизонт (1–N месяцев):
- 4 финансовых отчёта (Прибыли и убытки, Кэш-фло, Баланс, Использование прибыли);
- показатели эффективности инвестиций (NPV, IRR, PI, срок окупаемости и т.д.);
- оценку стоимости бизнеса (5 методов);
- финансовые коэффициенты (5 групп) и точку безубыточности;
- анализ рисков (чувствительность, Монте-Карло, What-If сценарии).

Дополнительно: мультиарендность (организации, роли), холдинги с консолидацией отчётов
нескольких проектов, биллинг (тарифы/квоты), план-факт (актуализация).

**Аудитория.** Финансовые аналитики, экономисты, инвестиционные менеджеры, финдиректора,
консультанты, владельцы МСБ. Люди работают с **плотными числовыми данными** и ждут
точности и контроля.

**Тон и характер.** Профессионально, по-взрослому, «банковская» надёжность + современный
продуктовый веб. Данные — герой. Это про **деньги**, поэтому: высокая аккуратность,
читаемость чисел, явные состояния, никакой «игрушечности».

**Язык и форматы.** Интерфейс — русский. Валюта — ₽. Числа — формат ru-RU (пробел —
разделитель тысяч, запятая — десятичная). Проценты — «15%». Месяцы обозначаются «М1…Мn».

**Темы.** Обязательны **светлая и тёмная** (переключатель в шапке). Обе — равноценные,
не «тёмная как доп.».

**Платформа.** Веб. **Desktop-first** (основная работа за большим экраном), но **адаптив
обязателен** до планшета и мобильного (просмотр результатов, лёгкое редактирование).

## 2. Дизайн-направление и референсы (прикладывать к каждому этапу)

**Стиль:** «современная финтех-аналитика». Чистая сетка, выразительная, но сдержанная
типографика, аккуратные плотные таблицы и графики, спокойная нейтральная палитра + 1–2
акцента, выверенные отступы, тонкие границы/тени, продуманные состояния и микровзаимодействия.

**Референсы по духу (НЕ копировать дословно):** Linear, Stripe Dashboard, Ramp, Mercury,
Pennylane, Causal, Runway, Vercel Dashboard, Notion. Для финмоделей ориентир на плотные,
но дышащие дата-таблицы и графики (как в Bloomberg-лайт, но дружелюбнее).

**Принципы:**
1. **Ясность важнее украшений.** Каждый пиксель служит данным или навигации.
2. **Данные — герой.** Числа — табличные цифры (tabular-nums), выравнивание по правому краю,
   акценты на итогах.
3. **Консистентность.** Единая система компонентов; одинаковые паттерны на всех экранах.
4. **Контроль и доверие.** Явные состояния сохранения, валидация, подтверждения, подсказки.
5. **Доступность.** Контраст ≥ WCAG AA, видимый фокус, навигация с клавиатуры, hit-targets ≥ 40px.
6. **Отзывчивость.** Skeleton-загрузки, оптимистичные апдейты, плавные, но быстрые переходы.

**Чего избегать:** перегруженный градиентами «дашборд ради дашборда», крупные пустые
герой-блоки в рабочих экранах, «облегчённые» нечитаемые числа, скрытие важного за ховером.

Каждый пункт должен быть выражен в макетах. Это контракт «функционал сохранён».

**Аутентификация и аккаунт**

- [ ] Создать пустой проект

**Редактор модели проекта (7 вкладок)**

      админ/произв./маркетинг/персонал×3; сумма помесячно; отсрочка; флаг «из прибыли»;
      флаг «валютная услуга»)
- [ ] Инвестиции (активы): название, стоимость, месяц покупки, срок службы, группа
      (земля/здания/оборудование), продажа (месяц+цена), переоценка (месяц+сумма)
- [ ] Финансирование: акционерный капитал (взносы: сумма+месяц); займы (сумма, старт, срок,
      ставка, погашение равными/в конце, «валютный», «проценты на прибыль»); лизинг
      (платёж, старт, срок, «финансовый»+ставка); депозиты/ЦБ (сумма, старт, срок, ставка);
      акции и дивиденды (число акций; дивиденды помесячно); автоподбор финансирования
      (вкл/выкл, ставка, мин. остаток)
- [ ] Валюта и старт: 2-я валюта (стартовый курс + курс помесячно); стартовый баланс
      (деньги, ОС, валютная позиция, дебиторка, кредиторка, сырьё, ГП, долг, капитал,
      нераспределённая прибыль) + индикатор сходимости актив=пассив
- [ ] Факт (актуализация / план-факт): вкл/выкл, «факт до месяца», ввод фактических данных
- [ ] Панель валидации (ошибки/предупреждения до расчёта)
- [ ] Защита несохранённых изменений (индикатор, предупреждение при уходе)
- [ ] Кнопки: ← Проекты, Анализ, **Рассчитать →**, Сохранить
- [ ] Помесячный ввод: сумма ряда «Σ», «заполнить все», вставка из Excel
- [ ] Подсказки «?» у неочевидных полей

**Результаты**
- [ ] Шапка: экспорт CSV / XLSX / Печать(PDF), ссылка «Анализ», версия движка
- [ ] Карточки показателей: NPV, IRR, PI, срок окупаемости, дисконт. окупаемость,
      потребность в финансировании (+ подсказки «?»)
- [ ] Карточки оценки бизнеса: чистые активы, по Гордону, DDM, по мультипликатору,
      ликвидационная (+ подсказки)
- [ ] Вкладки: Сводка (вердикт), Прибыли и убытки, Кэш-фло, Баланс, Коэффициенты, Графики,
      План-факт
- [ ] Сводка-вердикт: карточки со статусными цветами (создаёт ли стоимость, окупаемость,
      кассовый разрыв, ликвидность, накопленная прибыль, оценка) + предупреждения расчёта
- [ ] Таблицы отчётов: ОПУ (I1–I28), Кэш-фло (C1–C29), Баланс (B1–B34), с итоговыми

- [ ] Коэффициенты: 5 групп (ликвидность, оборачиваемость, структура капитала,

- [ ] Графики (6): денежный поток; накопленный поток (окупаемость); чистая прибыль;

- [ ] План-факт: факт по прошедшим периодам + отклонение (факт−план)

**Анализ рисков**
- [ ] Чувствительность: выбор параметра, набор коэффициентов → график NPV(параметр) + таблица

- [ ] What-If: сценарии (название + корректировки параметров) → сравнительная таблица

- [ ] Холдинг: состав (проекты + роль головной/дочерний), добавить/убрать проекты

**Организация и биллинг**
- [ ] Участники: список (email, ФИО, роль), добавить; роли (владелец/админ/редактор/наблюдатель)
- [ ] Тариф и оплата: текущая подписка (тариф, использовано/лимит проектов и участников);

**Сквозное**
- [ ] Светлая/тёмная темы; русский язык; ₽ и ru-RU числа
- [ ] Состояния: загрузка / пусто / ошибка — единообразные
- [ ] Тосты/обратная связь (сохранено, ошибка, скопировано)
- [ ] Печатная версия результатов (PDF)

# ЭТАП 1 — Дизайн-система (фундамент)

**Цель:** создать полноценную дизайн-систему, на которой строятся все экраны. Это самый
важный этап — заложи токены и компоненты, дальше всё переиспользуется.

**В составе:**
1. **Цвет.** Семантические токены для светлой и тёмной тем:
   - поверхности: `bg`, `surface`, `surface-2` (приподнятая), `border`;
   - текст: `text`, `text-muted`, `text-subtle`;
   - бренд/акцент: `primary` (+ hover, contrast);
   - статусы: `success`, `warning`, `danger`, `info` — каждый с вариациями `*-bg`,
     `*-border`, `*-text` (для алертов и в светлой, и в тёмной теме: в тёмной — полупрозрачный
     фон + светлый текст);
   - **палитра для графиков** (data-viz): 6–8 различимых цветов, работающих на обеих темах,
     различимых для дальтоников; + отдельные семантические (доход/расход, актив/пассив,
     положительный/отрицательный поток).
   Дай таблицу токенов со значениями hex/rgb для обеих тем и примеры контраста (AA).
2. **Типографика.** Семейство (предложи современный гротеск, напр. Inter / Geist / IBM Plex
   Sans), шкала (display, h1–h4, body, small, caption, mono для чисел), веса, межстрочные.
   Отдельно — **табличные цифры** (tabular-nums) для всех числовых данных.
3. **Сетка и отступы.** 8-pt система (4/8/12/16/24/32/48/64). Контейнеры, ширины контента,
   колоночные сетки. Брейкпоинты: mobile ≤640, tablet 641–1024, desktop 1025–1440, wide >1440.
4. **Радиусы, границы, тени.** Набор радиусов (sm/md/lg/pill), толщины границ, уровни
   elevation (в тёмной теме тени слабее — компенсируй границей).
5. **Иконки.** Стиль (line, 1.5–2px), сетка 20/24, набор под домен (проект, отчёт, график,
   деньги, процент, экспорт, настройки, организация, холдинг, предупреждение и т.д.).
6. **Motion.** Длительности (быстрые 120–200мс), easing, что анимируется (hover, появление
   панелей, переключение вкладок, тосты), и принцип «быстро и не отвлекает».
7. **Библиотека компонентов (атомы и молекулы)** — нарисуй каждый со всеми состояниями
   (default/hover/focus/active/disabled/error) в обеих темах:
   - кнопки (primary, ghost/secondary, danger, link-кнопка, иконочная), размеры, со спиннером;
   - поля ввода (text, number с шагом, с префиксом ₽/суффиксом %, с подсказкой «?», с ошибкой),
     select, checkbox, toggle, textarea, date;
   - **компонент помесячного ряда** (горизонтальная лента из N ячеек-инпутов с подписью,
     суммой «Σ» справа, кнопкой «заполнить все»; поддержка скролла при N=24; состояние
     «вставлено из Excel»);
   - **подсказка «?»** (бейдж + тултип);
   - вкладки (таб-бар, активная/неактивная, со счётчиком), сабнавигация;
   - карточки (метрика-карточка `m-label`+`m-value`; карточка-вердикт со статусной левой
     полосой; карточка-шаблон; карточка-тариф);
   - **таблицы данных** (плотная финансовая таблица: заголовок, строки, итоговые жирные
     строки, sticky-заголовок и sticky-первая-колонка, зебра/без, выравнивание чисел вправо,
     отрицательные — цветом/в скобках, hover-строка);
   - бейджи/чипы (статус, роль, тег «экспорт», «из прибыли», «валютный»);
   - алерты/баннеры (info/warning/danger), панель валидации (список замечаний);
   - тосты (успех/ошибка/инфо), модальные окна, подтверждения;
   - **состояния**: skeleton-загрузка (для списков, таблиц, карточек), пустое состояние
     (иллюстрация-заглушка + текст + действие), состояние ошибки;
   - графики: единая тема для столбчатых, линейных, area, стек-баров, пирога — оси, сетка,
     тултип, легенда, нулевая линия, реф-линии, подписи; поведение на тёмной теме.

**Адаптив:** покажи, как компоненты ужимаются (таблицы → горизонтальный скролл; формы →
1 колонка; помесячный ряд → скролл/свайп).

**Артефакты этапа:** страница «Foundations» (цвет/типографика/сетка/иконки/motion) +
страница «Components» (вся библиотека в светлой и тёмной темах, все состояния) + краткая
спецификация токенов в виде таблицы.

**Директива на проработку:** «Проработай дизайн-систему максимально детально и исчерпывающе.
Для КАЖДОГО компонента покажи все состояния (default/hover/focus/active/disabled/loading/error)
в светлой И тёмной темах, укажи размеры, отступы (в px и в токенах), радиусы, типографику,
поведение фокуса и доступность (контраст AA, hit-target). Не упускай ни одного элемента из
списка. Выдай токены таблицей. Это фундамент — используй максимум детализации.»

# ЭТАП 2 — Аутентификация (Вход / Регистрация)

**Цель:** первый контакт. Доверие, простота, бренд.

**Экраны:** `/login`, `/register`.

**Структура.** Центрированная карточка на нейтральном фоне (возможен сплит-лэйаут: слева
форма, справа — спокойный бренд-блок с кратким «что это» и преимуществами; на мобиле — только
форма). Логотип сверху. Под формой — переключение «Нет аккаунта? Регистрация» / «Уже есть? Вход».

**Поля.**
- Вход: email, пароль (с показать/скрыть), кнопка «Войти».
- Регистрация: email, пароль, ФИО, **название организации** (важно — при регистрации создаётся
  организация), кнопка «Создать аккаунт».

**Состояния:** пустое (default), фокус полей, ошибка валидации поля, ошибка сервера (баннер
«Неверный email или пароль»), загрузка (кнопка со спиннером, disabled), успех → редирект.

**Адаптив:** desktop сплит; tablet — карточка по центру; mobile — на всю ширину, крупные поля.
**Темы:** обе. **Директива на проработку:** «Покажи оба экрана в светлой и тёмной темах, на
desktop/tablet/mobile, со всеми состояниями полей и формы (default/focus/error/loading/успех),
вариант со сплит-бренд-блоком и без. Детализируй типографику, отступы, тексты ошибок, фокус.»

# ЭТАП 3 — Каркас приложения (App Shell) и навигация

**Цель:** общий каркас всех внутренних экранов: шапка, навигация, контейнер контента.

**В составе:**
- Шапка: логотип, основная навигация (Проекты/Холдинги/Организация — активный пункт),
  справа — селектор организации (дропдаран с ролью), переключатель темы (солнце/луна),
  email + меню пользователя («Выйти»).
- Контейнер контента (макс. ширина, отступы), хлебные крошки/контекст (где это уместно —
  напр. «Проекты / {название проекта} / Результаты»).
- **Вторичная навигация проекта:** заметный переключатель режимов
  **Редактор · Результаты · Анализ** (это ядро рабочего цикла) — предложи паттерн (сегмент-
  контрол в подшапке проекта или табы под названием проекта).
- Мобильная навигация: бургер/нижняя панель; как сворачивается шапка.

**Состояния:** активные пункты, hover, открытый дропдаран организации, одна организация
(селектор скрыт), длинные названия (обрезка).

**Директива на проработку:** «Спроектируй каркас детально: шапка (desktop/tablet/mobile),
все состояния навигации, селектор организации (1 и несколько), переключатель темы (свет/тьма),
меню пользователя, вторичная навигация проекта (Редактор/Результаты/Анализ) и хлебные крошки.
Покажи поведение при сужении до мобильного. Обе темы.»

# ЭТАП 4 — Проекты (домашняя) и шаблоны

**Цель:** старт работы. Список проектов + создание (пустого и из шаблона).

**Структура.**
- Заголовок «Проекты» + краткое описание.
- Блок создания: поле «Название», кнопка «Создать пустой»; ниже — **сетка из 4 карточек-
  шаблонов**: «Производство (демо)», «Торговля (опт)», «Услуги (консалтинг)»,
  «Действующее предприятие» — каждая с названием и кратким описанием (что внутри). Клик —
  создаёт проект из шаблона и открывает редактор.
- Список проектов: строки/карточки (название-ссылка, дата создания, кнопка «Удалить»).
  Подумай о более богатом представлении: можно показать мини-метрику проекта (если есть
  последний расчёт) — NPV/статус, но **не ломая** текущую модель (это опционально-визуальное).

**Состояния:** список (несколько проектов), **пусто** (нет проектов — крупный онбординг-блок:
«Создайте первый проект» + акцент на шаблоны), загрузка (skeleton-карточки), ошибка загрузки,
подтверждение удаления (модалка/инлайн), создание (кнопка со спиннером).

**Адаптив:** сетка шаблонов 4→2→1; список — карточки на мобиле.

**Директива на проработку:** «Детально проработай домашнюю страницу проектов: блок создания
(пустой + 4 шаблона-карточки с описаниями), список проектов (с действиями), все состояния
(список/пусто/загрузка-skeleton/ошибка/удаление-подтверждение). Предложи 2 варианта
представления списка (компактный список и карточки с мини-метрикой). Обе темы, адаптив 4/2/1.»

# ЭТАП 5 — Редактор проекта: каркас + вкладка «Проект» (настройки)

**Цель:** каркас редактора (общий для всех 7 вкладок) и первая, самая «настроечная» вкладка.

**Каркас редактора (используется на этапах 5–11):**
- Подшапка проекта: название проекта (редактируемое), переключатель **Редактор/Результаты/
  Анализ**, кнопки «← Проекты», «Анализ», **«Рассчитать →»** (акцентная).
- **Панель валидации** под названием: список замечаний — ошибки (красные) и предупреждения
  (жёлтые), напр. «Стартовый баланс не сходится: актив X ≠ пассив Y», «Темп роста g ≥ ставки
  дисконтирования — оценка по Гордону не посчитается». Сворачиваемая.
- **Таб-бар вкладок:** Проект · Сбыт · Издержки · Инвестиции · Финансирование · Валюта и старт ·
  Факт. (Подумай о подсказке-счётчике: сколько продуктов/издержек/активов заполнено.)
- **Save-bar (низ, липкая):** кнопка «Сохранить» (активна только при изменениях), индикатор
  «● несохранённые изменения» / «Сохранено ✓», ошибка сохранения. При уходе с несохранёнными —
  модалка подтверждения.

**Вкладка «Проект» — секции форм (сгруппировать визуально):**
1. *Проект:* Название; Дата старта (date); Длительность, мес. (number).
2. *Дисконтирование и оценка:* Ставка дисконтирования (год, %); Темп роста для оценки g (%)
   [подсказка: «должен быть меньше ставки дисконтирования»]; Мультипликатор прибыли (×)
   [0 — выключено]; Доля возврата при ликвидации (0–1).
3. *Налоги:* Налог на прибыль (%); Льгота по прибыли (доля 0–1); Страховые взносы с ФОТ (%);
   Налог на имущество (год, %); Налог с продаж (%).
4. *НДС и запасы:* НДС (%); Признание НДС (select: «По отгрузке» / «По оплате»); Оценка
   запасов ГП (select: «Средняя» / «ФИФО»); Производственный цикл, мес. [подсказка: «задержка
   между запуском и выпуском → НЗП»].
5. *Инфляция (год):* Цены сбыта (%); Прямые издержки (%); Зарплата (%); Общие издержки (%).

Используй компонент поля с подсказкой «?»; числовые — с суффиксами % / × / мес. Сгруппируй
секции карточками или разделителями с заголовками. Форма — 2–3 колонки на desktop, 1 на mobile.

**Состояния:** заполнено, пусто (значения по умолчанию), ошибки валидации (доля вне 0–1,
g≥ставки → предупреждение в панели), несохранённые изменения.

**Директива на проработку:** «Спроектируй каркас редактора (подшапка с переключателем
Редактор/Результаты/Анализ, кнопка Рассчитать, панель валидации с ошибками и
предупреждениями, таб-бар из 7 вкладок, липкий save-bar с индикатором несохранённых
изменений и подтверждением ухода) — он будет переиспользован на следующих этапах. Затем
детально — вкладку «Проект»: все 5 секций настроек со всеми полями (типы, суффиксы, подсказки
«?», валидация), 2–3 колоночная сетка, состояния (default/focus/error/несохранено), обе темы,
адаптив. Покажи панель валидации с примерами ошибки и предупреждения.»

# ЭТАП 6 — Редактор: вкладка «Сбыт»

**Цель:** ввод продуктов и плана продаж — здесь впервые появляется **помесячный ряд**.

**Структура.** Кнопка «+ Продукт». Список карточек-продуктов; пустое состояние:
«Добавьте продукт или услугу — объём и цена по месяцам формируют выручку».

**Карточка продукта:**
- Строка-шапка: название продукта (инпут), кнопка «Удалить».
- Помесячный ряд **«Объём»** (N ячеек, Σ, заполнить все, вставка из Excel).
- Помесячный ряд **«Цена»** (подпись меняется на «Цена (валюта)» при экспорте).
- Чекбокс **«Экспорт (во 2-й валюте, без НДС, по курсу FX)»**.
- Условия оплаты (3 поля + подсказки): «Предоплата (доля 0–1)» [→ авансы полученные],
  «Опережение предоплаты, мес.», «Отсрочка оплаты, мес.» [→ дебиторка].

**Состояния:** 0 продуктов (пусто), 1, много (прокрутка), длинный горизонт N=24 (ряд со
скроллом), экспортный продукт (валюта в подписи), ошибка предоплаты вне 0–1, фокус ячеек,
состояние «вставлено из Excel».

**Директива на проработку:** «Детально проработай вкладку Сбыт и особенно компонент
помесячного ряда (Σ, заполнить все, вставка из Excel, скролл при N=24). Покажи карточку
продукта со всеми полями и условиями оплаты с подсказками, экспортный вариант, пустое
состояние, несколько продуктов, ошибки. Обе темы, адаптив (на мобиле ряд — горизонтальный
скролл/свайп). Используй реалистичные числа (объём 100/мес, цена 1000 ₽).»

# ЭТАП 7 — Редактор: вкладка «Издержки»

**Цель:** прямые и постоянные издержки с типами, флагами, условиями оплаты.

**Две секции с «+»:**
1. **Прямые издержки** (пусто: «Материалы и сдельная оплата — формируют себестоимость
   продукции (COGS)»). Карточка: название; **тип** (select: Материалы / Сдельная зарплата);
   помесячный ряд «Сумма» (подпись «Сумма (валюта)» если валютный материал); поля «Отсрочка
   оплаты, мес.» [→ кредиторка] и (для материалов) «Опережающая закупка, мес.» [→ запас сырья],
   с подсказками; для материалов — чекбокс **«Валютный материал (по курсу закупки)»**.
2. **Постоянные издержки** (пусто: «Аренда, зарплата, реклама… — не зависят от объёма выпуска»).
   Карточка: название; **функция** (select: Административные / Производственные / Маркетинговые /
   Зарплата администрации / Зарплата производства / Зарплата маркетинга); помесячный ряд
   «Сумма»; «Отсрочка оплаты, мес.»; чекбоксы **«Из прибыли (невычитаемая)»** и
   **«Валютная (услуга, по курсу FX)»**.

**Состояния:** пусто (обе секции), смешанные типы, валютные/из-прибыли (видимые теги-чипы на
карточке?), ошибки, фокус.

**Директива на проработку:** «Проработай вкладку Издержки: обе секции (прямые/постоянные) со
всеми типами, флагами и условиями оплаты, подсказками; предложи компактные теги-индикаторы
на карточке (валютный / из прибыли / материалы-сырьё). Все состояния, пустые состояния, обе
темы, адаптив. Реалистичные числа (материалы 40 000 ₽/мес, аренда 15 000 ₽/мес).»

# ЭТАП 8 — Редактор: вкладка «Инвестиции» (основные средства)

**Цель:** активы с группировкой, амортизацией, продажей и переоценкой.

**Структура.** «+ Актив»; пусто: «Основные средства: капвложения, амортизация, продажа и
переоценка». Карточка актива: название; стоимость; месяц приобретения; срок службы (мес.);
**группа ОС** (select: Оборудование / Здания / Земля — у земли подсказка «без амортизации»);
чекбокс/раскрытие «Продаётся в течение проекта» (→ месяц продажи + цена продажи); опционально
«Переоценка» (→ месяц + сумма).

**Состояния:** пусто, оборудование/здание/земля, актив с продажей, актив с переоценкой,
несколько активов, валидация (месяц продажи ≥ покупки).

**Директива на проработку:** «Проработай вкладку Инвестиции: карточка актива со всеми полями,
группой ОС (земля/здания/оборудование, земля без амортизации — отрази визуально), опциями
продажи и переоценки (раскрывающиеся блоки), теги группы. Все состояния, пусто, обе темы,
адаптив. Реалистичные числа (станок 240 000 ₽, срок 24 мес).»

# ЭТАП 9 — Редактор: вкладка «Финансирование»

**Цель:** самая насыщенная вкладка — несколько секций источников финансирования.

**Секции (каждая с «+» и пустым состоянием-подсказкой):**
1. **Акционерный капитал:** взносы (сумма + месяц).
2. **Займы:** название; сумма; месяц получения; срок; ставка (год); погашение (select:
   Равными долями / В конце срока); чекбоксы «Валютный (по курсу FX)», «Проценты на прибыль
   (невычитаемые)».
3. **Лизинг:** название; платёж/мес; месяц начала; срок; чекбокс **«Финансовый лизинг
   (капитализация предмета → B19)»** (при включении появляется поле «Ставка лизинга (год)»).
4. **Депозиты / ЦБ:** название; сумма размещения; месяц; срок; ставка дохода (год).
5. **Акции и дивиденды:** число обыкновенных акций; помесячный ряд «Дивиденды».
6. **Автоподбор финансирования:** чекбокс «Покрывать дефицит наличности кредитной линией»;
   ставка (год); мин. остаток.

**Состояния:** все секции пустые, заполненные, финансовый vs операционный лизинг (разное
число полей), валютный заём (тег), много элементов (длинная вкладка — продумай навигацию/
якоря по секциям).

**Директива на проработку:** «Проработай вкладку Финансирование со всеми 6 секциями и всеми
полями/флагами; покажи раскрытие поля ставки при включении финансового лизинга, теги
(валютный/проценты на прибыль). Продумай навигацию по длинной вкладке (якоря/мини-оглавление
секций). Все пустые и заполненные состояния, обе темы, адаптив.»

# ЭТАП 10 — Редактор: вкладка «Валюта и старт» (стартовый баланс)

**Цель:** курс 2-й валюты и **детальный стартовый баланс с индикатором сходимости** — важный,
«бухгалтерский» экран.

**Структура.**
1. *Вторая валюта:* стартовый курс (число); помесячный ряд «Курс по месяцам»; пояснение
   «Пусто — одна валюта (без переоценки)».
2. *Стартовый баланс (действующее предприятие):* поля — Деньги, Остаточная стоимость ОС,
   Валютная позиция (ед. 2-й валюты), Дебиторка, Кредиторка, Запас сырья, Запас ГП,
   Долгосрочные займы, Акционерный капитал, Нераспределённая прибыль.
3. **Индикатор сходимости:** наглядно «Актив = Пассив» — зелёная плашка «Баланс сходится ✓»
   или красная «Актив − Пассив = X (должно быть 0, иначе расчёт не пройдёт)». Идеально — мини-
   визуализация двух чаш весов / двух столбиков актив vs пассив, обновляющаяся вживую.

**Состояния:** сходится (зелёный), не сходится (красный, с дельтой), пустой старт («с нуля»,
все нули — баланс сходится), одна валюта (курс пуст).

**Директива на проработку:** «Проработай вкладку Валюта и старт: блок курса (с помесячным
рядом) и стартовый баланс со всеми статьями. Особое внимание — индикатору сходимости актив=
пассив: спроектируй наглядную живую визуализацию равновесия (весы/два столбика), состояния
сходится/не сходится с дельтой. Обе темы, адаптив, реалистичные числа.»

# ЭТАП 11 — Редактор: вкладка «Факт» (план-факт / актуализация)

**Цель:** ввод фактических данных для сравнения план/факт.

**Структура.** Чекбокс «Включить актуализацию»; «Факт до месяца» (до какого периода есть факт);
ввод фактических значений по статьям/месяцам (таблица/ряды). Пояснение, что факт сравнивается
с планом на странице Результатов (вкладка План-факт).

**Состояния:** выключено (компактный onboarding «Включите, чтобы сравнивать план с фактом»),
включено (поля ввода факта по месяцам до «факт до месяца»), частично заполнено.

**Директива на проработку:** «Проработай вкладку Факт: переключатель актуализации, выбор
«факт до месяца», ввод фактических данных по периодам. Состояния выкл/вкл/частично. Свяжи
визуально с будущей вкладкой результатов «План-факт». Обе темы, адаптив.»

1. Дай Claude Design разделы **1–3** как постоянный контекст.
2. Запусти **Этап 1** (дизайн-система) — дождись токенов/компонентов.
3. Дальше **по одному этапу 2→20**, каждый раз прикладывая контекст + ключевые токены Этапа 1
   + блок «Директива на проработку» из этапа.
4. После сборки макетов вернись ко мне — свяжем экраны с backend по карте данных (Приложение A).
