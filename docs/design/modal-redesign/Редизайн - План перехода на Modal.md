# План перехода на дизайн «Modal — зелёный куб»
### с полным сохранением функционала и структуры «Финанс-Элит»

Источник: `uploads/DESIGN.md` — «Modal Style Reference: фосфорный терминал внутри голографического куба».
Цель документа: дать пошаговый план переноса визуального языка Modal на существующее приложение **без потери функционала**, **с сохранением переключателя Светлая/Тёмная тема** и с фирменным **зелёным кубом**.

> **Статус на текущий момент:** этапы A–G выполнены — все экраны 2–20 переведены на Modal, фундамент/пилот/куб-компонент готовы. Куб-система расширена: добавлен **Cube Hero** (насыщенная сцена — пол, звёзды, орбита частиц) для витринных моментов (вход, сплеш, приветствие после онбординга), поверх прежнего компактного/плоского куба для остальных мест. Из фазы **H (QA)**: сделаны `data-screen-label` на всех экранах, унификация радиусов (основные кнопки → 4px), рескинута типографика (алиас «Geist» → честный «Inter»), точечная проверка контраста Deep Reactor на светлой теме, старые до-Modal версии файлов перенесены в `Архив (до Modal)/`. Ещё не сделано: полный визуальный проход по обеим темам × 3 устройствам на каждом экране, и аудит дисциплины Reactor (см. ниже).

## 0. Что сохраняем · что перенимаем · что адаптируем

**Сохраняем без изменений**
- Весь функционал и структуру экранов (Этапы 2–20), всю логику, модель данных, расчёты и состояния (загрузка/пусто/ошибка/готово, тосты, подтверждения, RBAC, биллинг, консолидация и т.д.).
- **Переключатель темы Светлая / Тёмная — на каждом экране.**
- Каркас «панель управления + рамка устройства», адаптив desktop / tablet / mobile, русский язык, моноширинные числа.

**Перенимаем из Modal (новая визуальная ДНК)**
- Эстетику «фосфорного терминала»: дисциплинированная зелёная палитра, плоские поверхности, глубина — через стопку поверхностей, а не через тени.
- **Зелёный стеклянный куб** как фирменный объект и логотип (внутреннее свечение, лёгкий рассеянный «партикл»-фон).
- Острые радиусы (кнопки 4px, ссылки/инпуты 8px, карточки 16px) и **зелёно-тонированные нейтральные** (Fern / Lichen / Stone / Moss) вместо тёплых серо-бежевых.
- Типографику Goga (дисплей) / Inter (UI) — через доступные субституты.
- **Reactor Green `#7fee64` строго для действия / активного состояния** — никогда как декор.

**Адаптируем (здесь Modal расходится с нашими задачами — решаем осознанно)**
- Modal — тёмный лендинг; у нас — плотное продуктовое приложение с двумя темами. **Светлую тему выводим** из «Pale Mist» `#def0dd` (единственной светлой поверхности Modal), чтобы обе темы читались как один бренд. → Решение D1.
- Modal запрещает тёплые цвета. Но финансовому продукту **красный** (убыток / ошибка / удаление) и **янтарный** (предупреждение / близко к лимиту) несут смысл, на который опирается пользователь. Оставляем их — приглушённо и **только для семантики данных**. → Решение D2.

## 1. Ключевые решения (подтвердить до старта)

**D1 — Светлая тема.** Modal светлой темы не имеет.
- *Рекомендация:* перекоренить светлую тему в бледно-мятную гамму (канва из осветлённого Pale Mist, почти-чёрный мох для текста, зелёно-тонированные светлые границы). Так обе темы — один бренд.
- *Альтернатива:* оставить нынешнюю тёплую нейтральную светлую тему, переколеровать только тёмную. Менее цельно.

**D2 — Тёплые сигналы в финданных (красный/янтарь).** Modal: «никаких тёплых цветов».
- *Рекомендация:* сохранить **дисциплинированный приглушённый красный** (убыток, отрицательный поток, кассовый разрыв, удаление, ошибка валидации) и **янтарь** (предупреждение, близко к лимиту, черновик) — строго для смысла данных и системных статусов. Всё положительное / активное / декоративное — в зелёной системе. Это осознанный функциональный компромисс ради удобства.
- *Альтернатива:* строго по Modal — отрицательное показывать приглушением (Lichen/Stone) без красного. Чище по референсу, но ломает финансовую конвенцию.

**D3 — Контраст Reactor на светлом.** `#7fee64` — яркий, отлично работает как **заливка** (чёрный текст поверх), но **как текст/ссылка/иконка на белом не проходит контраст AA**.
- *Рекомендация:* ввести «**Deep Reactor**» (тёмно-зелёный, ~`#138A45`) для зелёного текста, ссылок и иконок на светлых поверхностях; заливки кнопок остаются на `#7fee64`.

**D4 — Шрифты.** Goga недоступен.
- Дисплей/заголовки: **Satoshi** или **General Sans** (ближе к «вырезанному» Goga), запасной — **Inter Display**. Веса 400–500, очень тугой трекинг.
- UI/текст: **Inter** (cv11), веса 400–700.
- Числа/код: **JetBrains Mono** (терминальный характер) — *рекомендация*; либо оставить **Geist Mono**, чтобы меньше менять.
- Примечание: Inter у нас обычно в «стоп-листе» как заезженный, но здесь он **оправдан** — это шрифт самого референса; компенсируем тугим трекингом, дисплей-фейсом и ss/cv-альтернатами.

**D5 — Зелёный куб.** Реализация и места появления.
- *Реализовано (`Куб.dc.html`):* **анимированный 3D CSS-куб** — медленное вращение (~19s), «дышащее» внутреннее ядро света, дрейф частиц, пульсирующее свечение-bleed; масштабируется под любой размер (container units), `prefers-reduced-motion` отключает анимацию. Проп `glow`: `true` — полный эффект (hero / вход / пустые состояния / CTA), `false` — спокойный плоский куб для навигации.
- Появляется: марка в навбаре/панели, экран входа (Этап 2), пустые состояния списков, загрузка/сплеш, момент «проект создаёт стоимость», финальные CTA. **Не злоупотреблять** — это единственный объект визуального веса.

**D6 — Порядок работы.** Сначала зафиксировать фундамент и пройти **пилот** на 2 контрастных экранах, получить апрув, затем раскатывать. (Детали — раздел 4.)

### 2.3 Типографика
- **Дисплей** (H1 экранов, hero, CTA): Satoshi/General Sans (или Inter Display), 400–500, трекинг −0.02…−0.03em. Заголовки «вырезанные».
- **UI/текст:** Inter (cv11), 400–700; тугой трекинг на мелком (−0.022em@14, −0.026em@12).
- **Eyebrow-метки** (надзаголовки секций): Inter 12px / 500 / UPPERCASE / трекинг **0.1em**, цвет Lichen или Reactor + точка-индикатор.
- **Числа:** Mono (JetBrains/Geist Mono); grand-итоги допустимо «фосфорно» подсвечивать Reactor.
- **Шкала** (из Modal): display 64 / lg 54 / heading 40 / sm 24 / sub 20 / body 16 / sm 14 / caption 12. В плотном UI заголовки экранов держим 24–40; 54–64 — только auth / CTA / маркетинг.

### 2.4 Форма и ритм
- **Радиусы:** кнопки **4px**, инпуты/ссылки **8px**, карточки **16px**, pill 9999px — только теги/бейджи. (Сейчас крупнее — станет острее.)
- **Spacing:** база 4px, секции 64–80px, паддинг карточек 16–20px, gap элементов 16px.
- **Тени:** тёмная — почти нет (стопка поверхностей); светлая — тонкие; **куб** — единственное исключение со свечением/градиентом.

### 2.5 Дисциплина акцента (важно)
Reactor Green **только** для: активная вкладка/навигация, основная кнопка, кольцо фокуса, «живые» индикаторы (идёт расчёт, баланс сведён), grand-итог.
**Не** для: декоративных плашек, крупных фонов, иконок «просто так», цепочек статус-бейджей. → заметно сократит текущее «эмеральд повсюду».

## 3. Спецификация компонентов (что меняется в общих паттернах)

- **Марка/логотип → зелёный куб** (нав 24px упрощённый; auth/пусто 96–160px со свечением; CTA — кластер).
- **Primary button:** заливка Reactor, текст чёрный, радиус 4px, без тени; hover — чуть ярче.
- **Ghost button:** прозрачный, 1px Fern, текст Phosphor Mint; hover — граница Reactor.
- **Inputs:** фон Void/на ступень выше, 8px, 1px Fern; focus — граница Reactor + кольцо. Состояния сохраняем.
- **Навбар / панель управления:** Carbon; активный пункт — нижняя граница Reactor 2px.
- **Вкладки (результаты/редактор):** активная — Reactor подчёркивание/левый бордер; неактивные — Lichen.
- **Карточки/поверхности:** база Carbon, граница Fern, без тени (тёмная), радиус 16.
- **Таблицы:** фон Carbon, линии gridLine (зелёно-тон.), закреплённая колонка Carbon, числа Mono Phosphor Mint; отрицательные — приглушённый коралл; grand-строка — Reactor/Soft Glow; ховер строки — Moss.
- **Чипы/бейджи:** pill; зелёно-тон. поверхности; статусы — Reactor (актив) / Lichen (нейтр) / коралл (проблема) / янтарь (внимание).
- **Модалки:** Carbon, граница Fern, радиус 16, оверлей чёрный; деструктив — primary в приглушённом коралле.
- **Тосты:** карточка Carbon, иконка-статус, без градиента.
- **Скелетоны:** Moss с pulse.
- **Графики:** палитра серий → [Reactor `#7fee64`, Soft Glow `#c8f9b6`, Sage `#aed2a4`, Lichen `#677d64`, + 1 коралл для «хуже плана»]; сетка gridLine, оси Lichen, тултип Carbon. На светлой — глубже.
- **Состояния:** куб в пустых; приглушённый коралл в ошибках (смысл сохраняем); скелетоны в загрузке.
- **Печать A4 (Этап 16):** бумага всегда белая; акценты — Deep Reactor/мох; отрицательные — в скобках + приглушённый красный (печать-friendly). Тему не трогаем.

## 5. Чек-лист правок в каждом `.dc.html`

1. Заменить блок `THEME()` (light+dark) на канонические палитры из раздела 2.
2. Шрифты: Geist → Inter (+ дисплей-фейс для заголовков), Geist Mono → моно по D4; добавить eyebrow-стиль.
3. Радиусы: кнопки 4 / инпуты-ссылки 8 / карточки 16 / теги pill.
4. Тени: убрать на тёмной (стопка поверхностей), оставить тонкие на светлой; куб — со свечением.
5. Логотип/марку → куб-компонент.
6. Аудит акцента: Reactor только на действии/активном; остальное — нейтраль.
7. Семантика: отрицательное/ошибка/деструктив → приглушённый коралл; предупреждение/лимит → янтарь.
8. Сохранить логику, состояния, `data-screen-label`, `data-comment-anchor`.

## 6. Риски и на что смотреть
- **Контраст Reactor на светлом** — решает Deep Reactor (D3).
- **«Inter = слоп»** — здесь оправдан (бренд-шрифт); смягчаем тугим трекингом, дисплей-фейсом, ss/cv-альтернатами.
- **Красный против запрета Modal** — осознанный функциональный компромисс (D2); держать приглушённым и только по смыслу.
- **Свечение/частицы куба** — следить за производительностью (лёгкий CSS, без тяжёлых фильтров на множестве узлов).
- **Числа на чистом чёрном** — Phosphor Mint, не белый; проверить читаемость и отсутствие «мерцания».
- **Объём (19 файлов)** — единый рецепт правок (раздел 5) + пилот снижают риск рассинхрона.
