# Декомпозиция работ: редизайн «Modal — зелёный куб» + связывание с backend

> Рабочий документ-чеклист реализации редизайна по пакету `design_handoff_modal_redesign/`
> (после P0 — `docs/design/modal-redesign/`). Ведётся исполнителем (Claude Code):
> чекбоксы отмечаются по мере выполнения, решения фиксируются здесь же.
>
> **Правила ведения:**
> - Один этап = один связный коммит (крупные этапы — несколько логичных коммитов).
> - После каждого этапа: `tsc --noEmit` + `npm run build` (frontend) и `pytest` (backend,
>   если менялся) — зелёные. Балансовые/golden-тесты движка не трогаем.
> - Контракты API не ломаем; расширения — только аддитивные (этапы B*).
> - Источник пиксельных значений — `renderVals()` соответствующего `.dc.html`;
>   палитра — **только** из `return THEME()` (см. Р1).

## 1. Сквозные технические решения (Р1–Р12)

- [ ] **Р1. Канон токенов.** Эффективная палитра всех экранов = объект, который **возвращает**
  `THEME()` (spread-override поверх легаси-базы; сама база мертва). Канонический список —
  `Modal Стиль-гайд.dc.html` → `THEME()` (помечен «copy into every screen») + расширенные
  ключи рабочих экранов (`subtotalBg, grandBg, hov, hovlab, cellBg, stickyBg, inputBg,
  ico1..4, success, violet→зелёные тона`). Примечание README о «каменной подложке» —
  устаревшее, игнорируем.
- [ ] **Р2. CSS-архитектура.** `styles.css` переводится на новый набор CSS-переменных,
  имена = ключи THEME (`--page-bg, --surface, --surface-2, --surface-3, --border,
  --border-strong, --grid-line, --text, --muted, --subtle, --primary, --primary-hover,
  --primary-text, --accent, --primary-bg, --primary-soft, --good*, --danger*, --warn*,
  --info*, --ring, --shadow, --menu-shadow, --bar-bg, --seg-bg, --seg-active-bg,
  --header-bg, --sticky-bg, --subtotal-bg, --grand-bg, --hov, --hovlab, --cell-bg,
  --input-bg, --sk`). Переключение тем — как сейчас, `[data-theme]` + `theme.ts`.
  Старые переменные удаляются в P13 (после перевода всех экранов).
- [ ] **Р3. Шрифты.** Inter (400–700, cv11) и JetBrains Mono (400–600) — self-host через
  `@fontsource/*` (npm). Satoshi (400/500/700) — v1 через Fontshare CDN (`<link>` в
  index.html), задача на self-host — в P13-backlog. Стеки: `--font-display:'Satoshi',
  'Inter',...`, `--font-ui:'Inter',...`, `--font-mono:'JetBrains Mono',ui-monospace,...`.
- [ ] **Р4. Графики — свои SVG, recharts удаляем.** Билдеры в макетах (Этапы 15/17,
  стиль-гайд) — готовый `React.createElement`-код: портируем в
  `components/charts/` (axisLayer, hoverBand, emptyEl + 6 графиков + sensitivity-línии,
  MC-гистограмма, спарклайны). После перевода Результатов и Анализа — `npm remove recharts`
  (≈ −380KB чанк). Палитры серий — из `PAL()` Этапа 15 (две темы).
- [ ] **Р5. CubeHero.** `components/CubeHero.tsx`: перенос разметки Cube Hero.dc.html в JSX
  (единицы cqmin сохраняем, container-type:size), RAF-цикл из макета (spin 32s/17s, bob,
  pointer-tilt lerp 0.07, intro 1.5s, у краёв — как в референсе). Пропсы как в макете:
  `accent=[c1,c2], backdrop, showEnvironment, showOrbit, motionSpeed, pointerTilt`.
  Дополнительно: `prefers-reduced-motion` → статичный кадр (без RAF и CSS-анимаций);
  в навбаре — environment/orbit выключены, pointerTilt выключен.
- [ ] **Р6. Иконки.** Инлайн-SVG из макетов → `components/icons.tsx` (набор: user, mail,
  lock, eye/eye-off, building, trash, copy, search, grid/rows, print, download, plus,
  chart, box, cart, briefcase, warning, check, arrow, burger…). Стиль 1.5–1.9 stroke.
- [ ] **Р7. Форматирование чисел.** `format.ts` расширить: `fmtTable(n)` — тысячи через
  NBSP, отрицательные «(1 234)», ноль «—» (fmt из Этапа 13); `fmtMoney` («12 480 000 ₽»,
  минус — «−»); `fmtAxis` («8,4м / 320к»); `fmtRatio` (запятая-десятичная, ед. ×/%/дн./₽);
  `fmtMillions` («+18,4 млн ₽»). Все числовые элементы — `--font-mono` + tabular.
- [ ] **Р8. Тосты и модалки.** Глобальный `ToastProvider` (портал; стек ≤3; авто-закрытие
  3с; типы success/info/warn/error — анатомия из Этапа 20) и компонент `Modal`
  (оверлей `rgba(0,0,0,.55–.7)` + blur, радиус 16, fe-pop; Esc закрывает, фокус-ловушка).
- [ ] **Р9. Состояния данных.** Единые компоненты: `Skeleton` (пульс, повторяет сетку
  контента), `EmptyState` (dashed-карточка/куб + заголовок Satoshi + CTA), `ErrorState`
  (иконка «!», текст, retry). Использовать на всех экранах (карта состояний — в этапах).
- [ ] **Р10. Проценты в формах.** Модель хранит доли (`0.20`), UI показывает «20 %».
  Конвертеры `fracToPct/pctToFrac` в одном месте (`format.ts`), применяются на всех
  полях-ставках (НДС, налоги, инфляция, ставки займов/лизинга/депозитов, дисконт, g,
  льгота, взносы, доля ликвидации, предоплата — предоплата и льгота остаются долями 0–1
  в UI, как в макете: см. подписи полей).
- [ ] **Р11. Данные и API-слой.** Сохраняем TanStack Query/AuthContext/axios; ключи
  запросов не меняем без нужды. Ошибки API → ErrorState/toast c `detail`.
- [ ] **Р12. Файлы макетов** в прод не попадают; служат референсом. Импортов из
  `docs/design/**` в `src/**` быть не должно (проверка в P13).

## 2. Матрица расхождений и принятые решения

### 2.1 Макет богаче backend → **делаем backend** (этапы B1–B5)

| Что в макете | Чего нет в API | Решение |

| Карточки проектов: NPV/IRR/окупаемость, статус «Рассчитан/Черновик» | В `ProjectSummary` только имя/даты | **B1**: хранить сводку последнего расчёта |
