## 0. Важное уточнение и правовая заметка

В репозитории **нет исходного кода**. Это **пиратская сборка коммерческого
desktop-продукта** Project Expert 7.21:

- `2 Project Expert 7.21/` — инсталлятор InstallShield (бинарники упакованы в `.cab`-архивы);
- `1 Подготовить компьютер/` — **эмулятор аппаратного ключа Hardlock** (`ВЗЛОМАТЬ.bat`
  подменяет драйвер `hardlock.sys` файлом-эмулятором `72b8.fst`). Это обход защиты.

**Выводы для нашего проекта:**

- ❌ **Нельзя** переиспользовать их бинарники, ресурсы (иконки, `pewres32.dll`), тексты
  справки, шаблоны отчётов и товарный знак «Project Expert» — это объекты авторского
  права и интеллектуальной собственности вендора (PRO-INVEST-IT / Expert Systems).
- ✅ **Можно** делать **«чистую» переразработку (clean-room)**: повторять *функционал* и
  использовать *общеотраслевые методики* финансового анализа (NPV, IRR, коэффициенты
  ликвидности и т.п. — они не охраняются авторским правом). Мы воспроизводим, *что*
  делает программа, не копируя, *как* написан её код.
- Файлы взлома (`ВЗЛОМАТЬ.bat`, `hardlock.sys`, `72b8.fst`, эмулятор) в наш продукт
  не входят и не нужны — мы заменяем всю модель лицензирования на подписку.

## 1. Что такое Project Expert

**Project Expert 7.21 Professional** — система **имитационного финансового моделирования**
предприятия.

- **Вендор:** PRO-INVEST-IT (позднее — Expert Systems / «Эксперт Системс»).
- **Копирайт:** © 1992–2001 (версия 7.x ~2001 года).
- **Назначение:** построить финансовую модель нового или действующего предприятия любой
  отрасли и масштаба, рассчитать денежные потоки и финансовую отчётность, оценить
  эффективность инвестиций и подготовить бизнес-план.
- **Целевые пользователи:** финансовые директора/менеджеры, инвест-аналитики,
  консультанты, банки (оценка проектов под кредит), разработчики бизнес-планов.

**Суть подхода — имитационная модель денежных потоков:**

- Хозяйственная деятельность представляется как непрерывная цепь преобразований активов,
  всегда проходящих через деньги; модель воспроизводит эти денежные потоки во времени.
- Расчёт **помесячный** (минимальный шаг — 1 месяц), горизонт — до десятков лет.
- Все величины — **временные ряды (векторы)** по месяцам; поверх них работает встроенный
  **язык формул**.
- Пользователь описывает сценарий (продажи, издержки, финансирование, налоги…),
  программа рассчитывает три формы отчётности и аналитические показатели; меняя
  параметры, можно «проигрывать» разные сценарии без риска.

> **Связь с нашим SaaS:** desktop OWL-приложение → web-приложение; монолитный бинарник →
> сервис расчётов + API; локальный файл `.pex` → облачная БД; аппаратный ключ → подписка.

## 4. Полная карта функционала

Рабочий процесс построен как набор разделов (из оглавления справки). 9 основных блоков:

### 4.1. Проект
- Заголовок/паспорт проекта (название, даты, длительность, автор).
- Список продуктов/услуг.
- Текстовое описание — дерево разделов бизнес-плана (структурированный текст).
- Отображение данных (настройки представления).
- **Настройка расчёта**: ставки дисконтирования (по каждой валюте), шаг дисконтирования
  (месяц/квартал/полугодие/год), параметры расчёта.
- Защита проекта (пароль).

### 4.2. Компания
- **Стартовый баланс** — начальное состояние действующего предприятия (~20 статей):
  акционерный капитал, добавочный капитал, денежные средства, нераспределённая прибыль,
  здания, земля, оборудование, НМА, незавершённые инвестиции, запасы готовой продукции,
  запасы комплектующих, счета к получению (дебиторка), счета к оплате (кредиторка),
  кредиты, полученные авансы, предоплаченные расходы, резервы, отсроченные налоги,
  ценные бумаги, продажа актива.
- Банк, система учёта (принципы бухучёта, периодичность).
- Структура компании (подразделения/бизнес-единицы).
- Разнесение издержек (распределение затрат по продуктам/подразделениям).

### 4.3. Окружение
- **Валюта** — мультивалютность (основная + вторая валюта), курс и динамика курса.
- Учётная ставка (ставка рефинансирования ЦБ).
- **Инфляция** — по группам объектов (сбыт, прямые издержки, недвижимость, зарплата,
  основные средства), помесячно/по годам.
- **Налоги** — настраиваемый налоговый блок: редактирование параметров, формулы расчёта,
  спец-обработка НДС и налога на прибыль, привязка к базам налогообложения.

### 4.4. Инвестиционный план
- **Календарный план** с **диаграммой Гантта**:
  - этапы (создание, редактирование, удаление, статус);
  - иерархическая организация этапов;
  - связи между этапами (предшествование), быстрое создание/редактирование связей;
  - этапы «Производство» (запуск выпуска продукта);
  - этапы с фиксированной датой;
  - информационное окно, шрифтовое оформление, доп. информация на диаграмме.
- **Список активов**: характеристики актива, амортизация, дополнительные инвестиции,
  переоценка активов, продажа актива, списание НДС.
- **Ресурсы**: использование ресурсов, описание платежей, диаграмма выплат за ресурсы.

### 4.5. Операционный план
- **План сбыта**: объём сбыта, ценообразование, скачкообразные изменения цены, условия
  оплаты, условия поставок, сложные схемы оплаты, варианты продаж, нестандартные налоги,
  быстрый ввод объёма, отражение в отчётах.
- **План производства**: продукты, материалы (рецептура/нормы расхода), сдельная
  зарплата, график производства, другие издержки, доп. возможности.
- **Сырьё, материалы и комплектующие**: общие данные, объём закупок, ценообразование,
  скачки цен, нестандартные налоги, отражение в отчётах.
- **План персонала**: штат, оклады, привязка к подразделениям, учёт.
- **Общие издержки**: административные/коммерческие/производственные накладные, привязка
  к продуктам, налоги/учёт.

### 4.6. Финансирование
- **Акционерный капитал** — выпуск акций, привлечение средств собственников.
- **Займы/кредиты**: поступление, возврат (график погашения), выплаты процентов,
  **автоподбор кредита** (программа сама подбирает график под дефицит наличности).
- **Лизинг**: лизинговые платежи, график выплат, выкуп, страхование, отражение в
  отчётности.
- **Инвестиции** (вложения в ценные бумаги/депозиты), **автоподбор инвестиций**.
- Другие поступления / другие выплаты.
- **Распределение прибыли** (дивидендная политика).
- Льготы по налогу на прибыль.

### 4.7. Результаты (выходная отчётность)
- **Отчёт о прибылях и убытках** (P&L).
- **Кэш-фло** (Cash Flow) — отчёт о движении денежных средств.
- **Баланс** (Balance Sheet).
- Отчёт об использовании прибыли.
- Детализация результатов (раскрытие строк).
- **Таблицы пользователя** — произвольные таблицы на встроенном языке формул.
- Графики и диаграммы (настройка графика, описание линий).
- Отчёт (компоновка выходного документа).
- **Экспертное заключение** — автоматически генерируемый текстовый вывод по проекту.

### 4.8. Анализ проекта
- **Финансовые показатели** (см. раздел 6).
- **Эффективность инвестиций**: PB, DPB, ARR, NPV, PI, IRR, MIRR (см. раздел 6).
- **Доходы участников**: денежные потоки участников, изъятие средств, эффективность для
  каждого участника.
- **Анализ чувствительности** — зависимость показателей (напр. NPV) от выбранных
  параметров; добавление варьируемых параметров.
- **Монте-Карло** — стохастическое моделирование: задание неопределённых параметров с
  диапазонами/распределениями, прогон множества итераций, статистика результата
  (устойчивость проекта, вероятностные оценки).
- **Анализ безубыточности** (break-even).
- **Анализ изменений** — сравнение версий проекта.
- **Доходы подразделений**.
- **Оценка бизнеса** — 5 методов:
  1. метод ликвидационной стоимости;
  2. метод предполагаемой продажи;
  3. метод чистых активов;
  4. метод экспертной оценки;
  5. **модель Гордона** (капитализация постоянного роста).
- **DDM** (Discounted Dividend Model) и мультипликаторы (оценка стоимости компании/акций).

### 4.9. Актуализация (план-факт)
- Ввод фактических данных по ходу реализации проекта.
- **Актуализированный Кэш-фло** (факт).
- **Рассогласование Кэш-фло** (анализ отклонений план/факт).

## 5. Встроенный язык формул

Ключевая особенность гибкости: пользователь строит произвольные таблицы и показатели на
**языке формул над временными рядами** (помесячными векторами). Состав (~40 функций):

- **Финансовые/временные:** Дисконтировать, Аккумулировать, Аккумулировать произведение,
  IRR, IRRV, Тенденция (тренд), Дифференцировать.
- **Агрегация и выборка:** Максимальные/Минимальные, Максимальное/Минимальное значение,
  Индекс максимального/минимального, Объединить, Объединить в период, Выбрать по периоду,
  Взять часть, Элемент.
- **Преобразования рядов:** Сдвиг, Повтор, Нормировать, Отклонение.
- **Математика:** Экспонента, Логарифм, Степень, Синус, Косинус, Модуль, Округлить,
  Остаток.
- **Логика и служебные:** Если (условие), Не пусто, Имя, Дата.

> **Связь с нашим SaaS:** это, по сути, предметно-ориентированный «табличный движок» над
> месячными рядами. В нашем продукте его стоит спроектировать как отдельную, хорошо
> протестированную подсистему формул (с безопасным парсером/вычислителем).

## 6. Аналитический контур (конкретные метрики)

### 6.1. Показатели эффективности инвестиций
Денежные потоки группируются в **инвестиции** (график потребности в капитале) и **чистые
поступления (CF)**; дисконтируются по ставке для каждой валюты.

| Показатель | Расшифровка |
|---|---|
| **PB** | Срок окупаемости (Payback Period) |
| **DPB** | Дисконтированный срок окупаемости (Discounted Payback Period) |
| **ARR** | Средняя норма рентабельности (Average Rate of Return) |
| **NPV** | Чистый приведённый доход (Net Present Value) |
| **PI** | Индекс прибыльности (Profitability Index) |
| **IRR** | Внутренняя норма рентабельности (Internal Rate of Return) |
| **MIRR** | Модифицированная IRR (Modified IRR, с нормой реинвестиций) |

### 6.2. Финансовые коэффициенты (5 групп)
Рассчитываются на основе консолидированных отчётов: Баланс (B), Отчёт о фин. результатах
(I), Отчёт об использовании прибыли (P); из баланса берутся средние за период значения.

- **Ликвидность (Liquidity):** коэффициент текущей ликвидности (Current Ratio),
  коэффициент срочной ликвидности (Quick Ratio), чистый оборотный капитал (Net Working
  Capital).
- **Деловая активность (Activity):** период оборачиваемости запасов, дебиторской
  задолженности (Average Collection Period), кредиторской задолженности
  (Creditor/Purchases Ratio); оборачиваемость рабочего капитала, основных средств (Fixed
  Assets Turnover), суммарных активов (Total Assets Turnover).
- **Структура капитала (Gearing):** Total Debt to Total Assets, Total Debt to Equity,
  коэффициент покрытия процентов.
- **Рентабельность (Profitability):** валовая маржа (Gross Profit Margin),
  рентабельность операционной прибыли, чистая маржа (Net Profit Margin), рентабельность
  активов (ROA), рентабельность собственного капитала (**ROE**).
- **Инвестиционные (на акцию):** прибыль на акцию (EPS), дивиденды на акцию, коэффициент
  покрытия дивидендов, сумма активов на акцию.

## 9. Модель лицензирования оригинала (и что меняем)

**Оригинал:** аппаратный ключ **Hardlock/HASP** в LPT/USB. Для локальной сети — менеджер
лицензий **HL-server** на машине с ключом, остальные клиенты получают лицензию по IP.
Утилита `run.exe` — для одновременной работы нескольких программ с однородными ключами.

**Наш SaaS:** полностью заменяем эту модель на **подписку**:

| Было (PE7) | Станет (наш SaaS) |
|---|---|
| Аппаратный ключ Hardlock + HL-server | Облачная подписка (тарифы, биллинг) |
| Лицензия на место по IP в LAN | Аккаунты/команды, роли, лимиты по местам и проектам |
| Однопользовательский desktop | Мультиарендность, совместная работа, шаринг |
| Файл `.pex` на диске | Проект в облачной БД (версии, автосохранение) |
| UI Borland OWL (2001) | Современный адаптивный web-UI |
| Word/Excel/DDE | PDF/XLSX/DOCX-экспорт + REST API + вебхуки |

**Что сохраняем 1:1 (ценность продукта):** помесячная модель денежных потоков; три формы
отчётности (P&L, Cash Flow, Баланс); налоги/инфляция/мультивалютность; метрики
эффективности (NPV/IRR/…); финансовые коэффициенты; анализ чувствительности / Монте-Карло
/ безубыточности; оценка бизнеса; язык формул; холдинг и what-if.
