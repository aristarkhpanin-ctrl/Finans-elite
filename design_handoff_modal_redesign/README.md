# Handoff: Редизайн «Финанс-Элит» → стиль «Modal — зелёный куб»

Пакет передачи дизайна для реализации в репозитории
`aristarkhpanin-ctrl/Finans-elite` (ветка `frontend/` — React 18 + TypeScript + Vite).
Продолжение разработки — в Claude Code.

---

## 1. Что это

Полный hi-fi редизайн веб-клиента «Финанс-Элит» (SaaS финансового моделирования) в
эстетике «Modal — зелёный куб»: чёрный/белый фон, ядовито-зелёный primary `#7FEE64`,
анимированный 3D-куб как брендовый герой, плотные финансовые таблицы, шрифты
Satoshi / Inter / JetBrains Mono.

Покрыты **все** экраны приложения: аутентификация, каркас/навигация, проекты, редактор
модели (Сбыт, Издержки, Инвестиции, Финансирование, Валюта, Факт), результаты (Сводка,
Таблицы, Коэффициенты, Графики), план-факт и печать, анализ рисков, холдинги,
организация/биллинг, а также сплеш-экран и гайд по состояниям/поведению.

## 2. Что такое файлы в этом пакете — ВАЖНО

Файлы `*.dc.html` — это **дизайн-референсы, а не production-код**. Это интерактивные
HTML-прототипы: каждый открывается в браузере двойным кликом (нужен `support.js` и
`Cube Hero.dc.html` рядом, они в пакете) и показывает точный внешний вид и поведение.

**Задача разработчика — воссоздать эти макеты в существующем стеке репозитория**
(React 18 + TS + Vite, собственная CSS-дизайн-система в `frontend/src/styles.css`),
используя уже существующие паттерны (react-router, TanStack Query, AuthContext,
Recharts, i18n). HTML-файлы напрямую в прод не переносятся.

Внутри каждого `.dc.html`:
- **разметка** — в блоке `<x-dc>…</x-dc>` (инлайн-стили, шаблонные дыры `{{ … }}`);
- **вся логика и стили** — в `<script data-dc-script>` класс `Component`:
  - `THEME()` — полные токены светлой/тёмной темы данного экрана;
  - `DIM()` — размеры для desktop / tablet / mobile;
  - `renderVals()` → объект `s` — готовые CSS-строки каждого элемента. **Это главный
    источник точных значений** (размеры, отступы, радиусы, тени, шрифты).

Верхняя панель каждого макета (сегменты «Тема / Устройство / Экран / Состояние») — это
**панель управления просмотром макета**, НЕ элемент продукта. В продукт из неё идёт
только сам факт наличия светлой/тёмной темы.

## 3. Фиделити

**High-fidelity.** Цвета, типографика, отступы, радиусы, состояния и микро-анимации —
финальные. Воспроизводить пиксель-в-пиксель, копируя значения из `renderVals()`.

### Типографика
- **Satoshi** (Fontshare: `https://api.fontshare.com/v2/css?f[]=satoshi@400,500,700`) —
  дисплей: H1 экранов, hero-заголовки, титулы отчётов. Вес 500/700, letter-spacing −.02…−.025em.
- **Inter** (Google Fonts, 400/500/600/700) — весь UI: кнопки, поля, лейблы, таблицы.
- **JetBrains Mono** (Google Fonts, 400/500/600) — ВСЕ числа: ячейки таблиц, KPI, чипы, суммы.
- Базовые размеры: UI 12–14.5px; лейблы-капсы 9.5–12px (`letter-spacing:.06–.08em; uppercase`);
  H1 21–31px (Satoshi); числа таблиц 12–13px mono. Точные значения — в `renderVals()` экрана.

### Форма и скругления
- Кнопки: радиус **4px**, высота 34–48px; primary — зелёная заливка, чёрный/тёмный текст.
- Поля ввода: радиус 8px, бордер 1.5px, фокус-кольцо `0 0 0 3px ring`.
- Карточки: радиус 14–16px; сегмент-контролы: обёртка 9px / кнопка 7px.
- Чипы/бэйджи: радиус 9999px (pill), шрифт mono 11px.
- Модалки: радиус 16px, оверлей `rgba(0,0,0,.55–.7)` + blur 2–4px.

## 6. Cube Hero (брендовый куб)

`Cube Hero.dc.html` — анимированный 3D-куб (CSS 3D transforms, ~зелёное свечение,
медленное вращение ~14s, лёгкий float). Воссоздать как `components/CubeHero.tsx`.

Пропсы макета (сохранить API): `backdrop` (`'transparent'` для встраивания),
`show-environment` (пол/свечение), `show-orbit` (орбитальные частицы), size — от контейнера.

Где используется (везде через встраивание, анимация всегда включена — решение заказчика):
- навбар всех экранов — 26px (environment/orbit выключены);
- сплеш-экран — крупный по центру;
- Этап 2: брендовая панель сплита (34px марка + большой hero), полупрозрачный за
  галочкой успеха;
- Этап 4: эмпти-стейт «Создайте первый проект» (46px) и оверлей «Проект создан»;
- Этап 12: акценты карточек KPI; Этап 17: лоадер Монте-Карло.

## 7. Ключевые паттерны поведения (см. Этап 20 — полный гайд)

- **Темы**: обе темы во всех экранах; переключатель в шапке (`theme.ts` уже есть).
  Тёмная — дефолт витрины, светлая — дефолт рабочей области (уточнить у заказчика).
- **Загрузка**: skeleton-плейсхолдеры (`sk` токен), спиннер `border-top-color` 0.7s.
- **Пустые состояния**: карточка с dashed-бордером, куб/иконка, заголовок Satoshi, CTA.
- **Ошибки форм**: бордер danger + shake 0.4s; баннер сервера над формой (Этап 2).
- **Успех**: зелёный круг с галочкой, `fe-pop .4s`, прогресс-бар редиректа 1.6s.
- **Оверлей «Проект создан»** (Этап 4): радиальный градиент фона
  тёмный `#16331D→#0A0D09` / светлый `#DBF3D2→#F4FAF2`, куб, Satoshi-заголовок.
- **Таблицы**: липкий первый столбец, mono-числа, итоговые строки `primarySoft`,
  отрицательные — danger, hover строки `surface2`.
- **Модалки/тосты/меню**: `menuShadow`, появление `fe-fade/fe-pop ~0.25–0.35s`.
- **Брейкпоинты макетов**: desktop 1080–1240px, tablet 864px, mobile 402px
  (радиус рамки 26–30px). В продукте — адаптив по этим же трём режимам.

## 8. Состояние данных

Использовать существующие: `AuthContext` (JWT, X-Organization-Id, авто-выход при 401),
TanStack Query для проектов/результатов, `PUT` сохранение модели, lazy-чанк результатов.
Редизайн не меняет контракты API — только представление.

## 10. Как использовать в Claude Code

1. Положить папку в репозиторий, например: `docs/design/modal-redesign/`.
2. Коммит: `git add docs/design && git commit -m "Add Modal redesign handoff"`.
3. В `CLAUDE.md` репозитория добавить строку:
   > Дизайн-референсы редизайна — в `docs/design/modal-redesign/` (README.md — маппинг
   > макетов на файлы `frontend/src/`, токены и правила). Макеты `.dc.html` — эталон
   > внешнего вида; точные стили — в `renderVals()` внутри каждого файла.
4. Стартовый промпт для Claude Code:
   > Прочитай docs/design/modal-redesign/README.md. Реализуй редизайн поэтапно:
   > сначала дизайн-токены в frontend/src/styles.css и компонент CubeHero, затем
   > экраны по маппингу из README (начни с Layout и LoginPage). Пиксель-в-пиксель
   > по значениям из renderVals() соответствующего .dc.html.
