# backend — расчётное ядро (`calc_core`)

Изолированное расчётное ядро финансовой модели (этап 4). Чистая детерминированная
функция `run(model) -> CalcResult` без зависимостей от БД/HTTP/арендаторов
(см. [`../docs/ARCHITECTURE-SaaS.md`](../docs/ARCHITECTURE-SaaS.md) §6 и
[`../docs/CALC-ENGINE-SPEC.md`](../docs/CALC-ENGINE-SPEC.md)).

## Что реализовано (`engine_version 0.6.0`)

Согласованный (балансирующийся) срез функционала:

- **Модели проекта** (Pydantic v2) — дерево входных данных, замена `.pex`.
- **Отчёты** с точными кодами и метками строк: ОПУ (`I1–I28`), Кэш-фло (`C1–C29`),
  Баланс (`B1–B34`), Использование прибыли (`P1–P7`) — итоги по формулам спецификации.
- **Движок**: сбыт, прямые/постоянные издержки, амортизация, налог на прибыль и
  имущество, займы (проценты + тело), акционерный капитал, дивиденды.
- **Оборотный капитал (5.1a):** условия оплаты → дебиторка (`B2`), кредиторка (`B23`),
  авансы (`B24`) — первый «честный» разрыв accrual/cash.
- **Запасы (5.1b):** опережающая закупка → сырьё (`B3`); производство ≠ сбыт → готовая
  продукция (`B5`) по средней себестоимости; себестоимость (`I5`/`I6`) признаётся при
  продаже. Капитализация затрат сглаживает валовую прибыль к продажам.
- **НДС (5.1c):** Кэш-фло с НДС, ОПУ без НДС; исходящий НДС с продаж, входной — с закупок,
  материалов, общих издержек и capex; зачёт «по отгрузке», НДС к уплате в `C12`, НДС-кредит
  переносится в `B7`.
- **Автоподбор финансирования (7.1):** кредитная линия покрывает дефицит наличности до
  `min_balance`; проценты влияют на прибыль/налог → расчёт итеративный (сходится).
  Краткосрочный долг → `B22`. Гарантия: деньги ≥ `min_balance`.
- **Показатели эффективности** (предварительно): NPV, IRR, PI, срок окупаемости.
- **Финансовые коэффициенты (5.2):** 5 групп — ликвидность, деловая активность, структура
  капитала, рентабельность, инвестиционные (на акцию); деление на ноль → `None`.
- **Инварианты** (SPEC §16): `B20 = B34`, `B1 = C29`, `B32 = ΣP7` — проверяются на каждом
  расчёте; нарушение = `InvariantError`.

### Под-части 5.1 (наращивание ядра) — завершено
- ✅ **5.1a** — оборотный капитал по условиям оплаты (дебиторка/кредиторка/авансы).
- ✅ **5.1b** — запасы: сырьё `B3` и готовая продукция `B5` (НЗП `B4` — отложено,
  требует модели длительности производственного цикла).
- ✅ **5.1c** — НДС (Кэш-фло с НДС, ОПУ без НДС, НДС к уплате в `C12`, НДС-кредит в `B7`).

> Каждая под-часть добавлена **с сохранением балансового инварианта** `B20 = B34`.
>
> Упрощения v0 (к валидации по эталону, SPEC §22): НДС только «по отгрузке»; прямые
> налоги и НДС к уплате — в периоде начисления (без лага расчётов с бюджетом); НДС с
> авансов отдельно не начисляется; единая ставка НДС (без частного НДС по продуктам).

### Тарифы и квоты (6.5a)
| Тариф | Цена ₽/мес | Проекты | Участники |
|---|---|---|---|
| free | 0 | 5 | 5 |
| team | 2900 | 50 | 25 |
| business | 9900 | ∞ | 100 |

Каждая организация получает подписку (по умолчанию `free`). Создание проекта/добавление
участника сверх лимита → `402 Payment Required`. Смену тарифа выполняет владелец
(право `billing.manage`): прямая (`/subscription`) или через оплату (`/billing/checkout`).

**Платежи (6.5b):** провайдер выбирается по окружению — при заданных `YOOKASSA_SHOP_ID`
и `YOOKASSA_SECRET_KEY` используется ЮKassa (иначе — ручной провайдер, мгновенная смена).
`/billing/checkout` создаёт платёж с чеком 54-ФЗ и возвращает ссылку оплаты; вебхук
`payment.succeeded` активирует тариф (идемпотентно, обрабатывается только известный платёж).
Прод-харднинг вебхука (IP-список ЮKassa / повторный запрос статуса через API), реальные
ключи и `YOOKASSA_VAT_CODE` — через переменные окружения.

### Роли и права (6.4)
Иерархия (права накапливаются): **viewer ⊂ analyst ⊂ editor ⊂ admin ⊂ owner**.

| Роль | Права |
|---|---|
| viewer | читать проекты, считать, видеть участников |
| analyst | + создавать/редактировать проекты |
| editor | + удалять проекты |
| admin | + управлять участниками |
| owner | + организация и биллинг |

## HTTP-API (5.3)

FastAPI-приложение вокруг ядра (`app/`). Эндпоинты:

| Метод | Путь | Назначение |
|---|---|---|
| `GET` | `/health` | живость + версия методики |
| `GET` | `/api/v1/sample` | демо-модель проекта (готова к отправке в `/calculate`) |
| `POST` | `/api/v1/calculate` | расчёт без сохранения: отчёты, показатели, коэффициенты |
| `POST` | `/api/v1/auth/register` | регистрация (пользователь + организация) → токен |
| `POST` | `/api/v1/auth/login` | вход по email/паролю → токен |
| `GET` | `/api/v1/auth/me` | текущий пользователь |
| `POST` | `/api/v1/organizations` | создать организацию (создатель — владелец) |
| `GET` | `/api/v1/organizations` | мои организации (с ролью) |
| `GET` | `/api/v1/organizations/{id}` | данные организации |
| `POST` `GET` | `/api/v1/organizations/{id}/members` | добавить / список участников |
| `GET` | `/api/v1/plans` | каталог тарифов (публично) |
| `GET` `POST` | `/api/v1/organizations/{id}/subscription` | подписка / прямая смена тарифа |
| `POST` | `/api/v1/organizations/{id}/billing/checkout` | инициировать оплату тарифа (ЮKassa) |
| `POST` | `/api/v1/billing/webhook/yookassa` | вебхук ЮKassa (активация по оплате) |
| `POST` `GET` | `/api/v1/projects` | создать / список проектов (в организации) |
| `GET` `PUT` `DELETE` | `/api/v1/projects/{id}` | получить / обновить / удалить проект |
| `POST` | `/api/v1/projects/{id}/calculate` | рассчитать сохранённый проект |
| `GET` | `/docs`, `/openapi.json` | авто-документация OpenAPI |

Эндпоинты организаций и проектов требуют **`Authorization: Bearer <token>`** (JWT из
`register`/`login`). Организация запроса — первая организация пользователя; для выбора из
нескольких — заголовок `X-Organization-Id` (проверяется членство). Несходящийся стартовый
баланс/ошибка валидации → `422`; нет доступа → `403`; не найдено → `404`.

**Хранение (6.1) / мультиарендность (6.2) / аутентификация (6.3):** SQLAlchemy 2.0
(SQLite в dev/test, PostgreSQL в проде через `DATABASE_URL`, JSON → `JSONB`); проекты
изолированы по `organization_id`; пароли — Argon2; токены — JWT (`JWT_SECRET`,
`JWT_TTL_SECONDS` из окружения).

```bash
uvicorn app.main:app --reload     # запуск сервера, документация на /docs
```

## Структура

```
backend/
├── pyproject.toml
├── calc_core/
│   ├── money.py          # Decimal-арифметика
│   ├── series.py         # помесячные ряды
│   ├── metrics.py        # NPV/IRR/PI/окупаемость
│   ├── version.py        # ENGINE_VERSION
│   ├── samples.py        # демо-проект
│   ├── demo.py           # python -m calc_core.demo
│   ├── models/           # Pydantic-модели проекта
│   ├── reports/          # строки, отчёты, коэффициенты, результат
│   └── engine/           # конвейер расчёта + инварианты
├── app/                  # FastAPI-приложение
│   ├── main.py           # сборка приложения, lifespan, расчётные роуты
│   ├── schemas.py        # схемы ответов, проектов, организаций
│   ├── database.py       # подключение к БД (SQLAlchemy)
│   ├── db_models.py      # ORM: Organization, User, Membership, Project
│   ├── crud.py           # CRUD (изоляция проектов по organization_id)
│   ├── security.py       # хэш паролей (Argon2) + JWT
│   ├── rbac.py           # роли и матрица прав
│   ├── plans.py          # каталог тарифов
│   ├── billing.py        # текущий тариф, квоты, PaymentProvider, выбор провайдера
│   ├── payments_yookassa.py  # клиент и провайдер ЮKassa (54-ФЗ, вебхук)
│   ├── deps.py           # зависимости (пользователь, организация, проверка прав)
│   └── routers/          # auth, billing, organizations, projects
└── tests/                # юнит + инвариант + API + персистентность + мультиарендность

```

## Запуск

```bash
cd backend
python -m pip install -e ".[dev]"   # или: pip install pydantic fastapi "uvicorn[standard]" pytest httpx
pytest -q                           # тесты
python -m calc_core.demo            # демонстрационный расчёт (CLI)
uvicorn app.main:app --reload       # HTTP-API, документация на /docs
```
