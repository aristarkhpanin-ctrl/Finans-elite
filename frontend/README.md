# frontend — веб-клиент (React + TypeScript + Vite)

Веб-интерфейс для SaaS финансового моделирования. Этап Ф1: каркас и аутентификация.

## Реализовано
**Ф1 — каркас и аутентификация:**
- Vite + React 18 + TypeScript; собственная дизайн-система на CSS (светлая/тёмная тема).
- Маршрутизация (react-router): публичные `/login`, `/register`; защищённые под `Layout`.
- Аутентификация (JWT): контекст `AuthContext`, axios-клиент с подстановкой токена и
  `X-Organization-Id`, авто-выход при 401; выбор организации, переключатель темы, i18n (RU).

**Ф2 — проекты и редактор модели:**
- Список проектов: создание, удаление, открытие (TanStack Query + мутации).
- Редактор модели с вкладками: Проект (заголовок/настройки), Сбыт (продукты, помесячные
  объём/цена, условия оплаты), Издержки (прямые/постоянные), Инвестиции (активы),
  Финансирование (капитал, займы, автоподбор).
- Переиспользуемый ввод помесячных рядов (`MonthlySeries`); сохранение через `PUT`.

**Ф3 — результаты:**
- Кнопка «Рассчитать» (сохранить + перейти к результатам).
- Карточки показателей (NPV/IRR/PI/окупаемость), под-вкладки: ОПУ, Кэш-фло, Баланс,
  Коэффициенты, Графики.
- Плотные финансовые таблицы (липкий столбец статей, помесячные колонки, выделение итогов).
- Графики (Recharts): денежный поток и чистая прибыль. Страница результатов — отдельный
  чанк (lazy-load), recharts не утяжеляет основной бандл.

**Ф4 — анализ:**
- Чувствительность: параметр + коэффициенты → график NPV и таблица.
- Монте-Карло: неопределённые параметры (равномерное/нормальное) + N/seed → статистика
  NPV и вероятность NPV>0.
- What-If: сценарии с корректировками → сравнение (график и таблица NPV/IRR/PI/окупаемость).

**Ф5 — организация и биллинг:**
- Участники: список с ролями, добавление по email с ролью (ошибки прав/квот — понятные).
- Тариф и оплата: текущая подписка с использованием квот, каталог тарифов, переход на
  тариф через checkout (ЮKassa-redirect или мгновенно у ручного провайдера).
- Навигация Проекты / Организация в шапке.

**Ф6 — расширенный редактор (вывод возможностей ядра):**
- Настройка расчёта: признание НДС (по отгрузке/оплате), оценка запасов (средняя/ФИФО),
  льгота по прибыли, страховые взносы с ФОТ, инфляция по 4 группам.
- Сбыт: флаг «экспорт» (во 2-й валюте, без НДС). Издержки: флаги «из прибыли» и «валютная».
- Инвестиции: продажа актива (месяц + цена). Финансирование: валютные займы и проценты
  «на прибыль», разделы «Лизинг» и «Депозиты/ЦБ».
- Вкладка «Валюта и старт»: курс 2-й валюты (старт + помесячный ряд), валютная позиция и
  полный стартовый баланс с индикатором сходимости.

**Ф7 — холдинги (консолидация):**
- Раздел «Холдинги»: создание/удаление холдинга, добавление проектов с ролью
  (головная/дочерняя).
- «Консолидировать» → сводный бюджет (NPV/IRR/PI + ОПУ/Кэш-фло/Баланс/Коэффициенты)
  через Integrator; переиспользует таблицы отчётов.

**Ф8 — результаты+ и план-факт:**
- Графики-долёвки: круговая диаграмма структуры издержек за весь период.
- Актуализация (план-факт): вкладка редактора «Факт» (ввод факта по строкам Кэш-фло) и
  вкладка результатов «План-факт» (актуализированный Кэш-фло + отклонение факт − план).
- Экспорт отчётов в CSV (ОПУ/Кэш-фло/Баланс/Использование прибыли).

## Запуск
```bash
cd frontend
npm install
npm run dev        # http://localhost:5173 (прокси /api → backend :8000)
npm run build      # production-сборка
npm run typecheck  # проверка типов
```
Backend должен быть запущен на :8000 (или задайте `VITE_API_URL`).
